# 目的

- 「テックブログを書いて」という指示が来たときの最短ワークフローを定義する
- 既存 docs/ の流儀に合わせて新規記事を追加し、目次も更新する

# 使う場面

- ユーザーが「techblog 書いて」「テックブログ書いて」「ブログにして」など、記事作成を指示したとき

# 進め方

1) docs/README.md を開いて命名規則と目次の番号体系を確認する  
2) 既存の docs/ 記事を1つだけ見て文体・構成を合わせる  
3) 次の連番で新規 md を作成（例: 10-xxx.md）  
4) 必要な内容を **結論→理由→補足→まとめ** の順で書く  
5) docs/README.md に目次を追記する  

# 記事の書き方（必須）

- 冒頭は必ず `AIが書いたテックブログ` を入れる  
- 1つ目の見出しは結論（太字を含む）  
- 具体例や条件分岐を入れて「いつ起きるか」を明示する  
- 余計な煽りや雑談は入れない  
- 500〜900字程度にまとめる  

# 目次の追記ルール

- docs/README.md の `# 目次` の箇条書きに追加  
- 並び順は番号順  

# 注意

- 既存の番号と重複させない  
- 記事のタイトルは短く明確に  
- 日本語で書く
